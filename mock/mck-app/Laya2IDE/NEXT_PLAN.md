# Laya2IDE 后续演进与开发计划 (Next Phase Plan)

基于项目前期的重构方案及 `PLAN.md` 中的历史实施进度，Laya2IDE 已经完成了基础设施（Zustand Store）、渲染解耦（双 Canvas 架构）以及初步的交互支持（选中、独立私有属性、拖拽修改等）。

本文档对尚未完成的任务进行了梳理归纳，并在此基础上提供了进一步的长期演进架构建议。

---

## 1. 遗留任务梳理 (Pending Tasks)

以下功能在之前的 Phase 4 计划中被提出，但目前尚未完全落地，属于下一阶段开发的优先事项：

### 1.1 层级树 (Hierarchy) 补齐
*   **右键菜单补全**：右键菜单已支持 `CreateImageUI`，需补充实现 `Delete`（删除当前节点及其子树）、`Duplicate`（克隆完整节点及属性），以及其他基础类型的创建（如 Node、Sprite、Label）。
*   **动作一致性**：将“添加”、“删除”、“调整层级顺序”等影响 Scene 数据树结构的动作全部封装统一的更新修改器（Mutator/Command），确保计算好如 `hasChild` / `isDirectory` 等级联和派生字段。

### 1.2 属性面板 (Inspector) 会话与重构
*   **动态基于 Schema 的表单**：废弃根据 type 硬编码判断字段的方式，将 Inspector 升级为通用的 schema 动态表单系统（如按组件和节点类型组合渲染对应的调节块）。
*   **UI 面板分组**：视觉上将 Inspector 拆分为“基础属性组（Transform）”、“类型专属属性组”与“底层私有属性组”，提供清晰且折叠的层级。
*   **背景管理入口**：在 Inspector 中补充专门针对根节点 `sceneBg` 的可视化编辑输入功能，附带“手动重新加载”动作按钮。
*   **私有属性完善**：面板内增加一键“重置默认值”按钮，同时在本地存储数据结构中加上 `version`，以便未来字段变更时无缝升级旧版本结构。

### 1.3 核心交互行为优化
*   **拖拽阈值（死区）**：引入 2~4px 的拖拽阻尼（阈值），避免用户单纯点击（Click）时的微小手抖被判定为拖拽修改。
*   **坐标轴向锁定**：支持在拖动物体时按下 `Shift` 键，锁定位移为纯水平（X轴）或纯垂直（Y轴）方向。
*   **异步资源刷新**：在 Inspector 修改 `skin` 或 `texture` 字段失焦后，应能够主动触发事件，重新调用图片预加载方法以完成 UI 内容刷新，避免出现需要重载整个应用的体验断层。
*   **报错与监控可视**：背景图、节点引用图片加载失败时，应当在界面底部状态栏（或独立 Console 视图）输出明确日志与 Warning。
*   **历史链路梳理**：对于“鼠标拖拽”和“键盘方向键移动记录”，统一归拢到统一会话（Session ID + 合并策略）机制，保障连续动作只占用一个 Undo 动作，并在底部状态栏展示诸如 `History: [cursor]/[total]` 以协助排查记录健康度。

### 1.4 资源管理器 (Asset Explorer) 当前落地状态（2026-04）
*   ✅ 已完成底部资源管理器独立组件化（`AssetExplorerPanel`）。
*   ✅ 已完成左右分区：
	- 左侧为当前工程资源。
	- 右侧为外部导入资源文件夹（支持累加导入多个目录）。
	- 当右侧无导入目录时，左侧自动铺满。
*   ✅ 已完成“当前工程 assets 根目录自动识别”：
	- 从当前打开的 scene 文件路径向上遍历，找到 `Laya/laya` 目录后，定位其下 `assets` 目录。
*   ✅ 已完成左侧目录下拉切换：
	- 默认项为 `assets`。
	- 自动扫描当前 scene 中 `Image/Sprite` 的 `skin` 字段，将对应 png 所在目录动态加入下拉项。
*   ✅ 已完成图片网格预览：
	- 同尺寸缩略图渲染（统一卡片尺寸）。
	- 下方显示“文件名（不含后缀）+ 后缀”。
*   ✅ 已支持混合资源项显示：
	- 不仅显示图片，也显示文件夹和其他类型文件。
	- 非图片文件使用统一文件图标占位渲染。
*   ✅ 已支持目录浏览交互：
	- 双击文件夹可进入子目录。
	- 自动渲染 `..` 条目，双击可回到父目录。
	- 浏览范围受根目录限制，不会越界到根目录外。
*   ✅ 已支持当前路径面包屑导航：
	- 工程资源区会在标题行右侧显示 `assets/...` 面包屑，可点击任一层级快速跳转。
	- 外部资源区同样具备面包屑导航，便于快速回退与切换。
*   ✅ 已支持“临时路径下拉项”机制：
	- 当当前目录不在原始下拉选项时，会自动追加到下拉末尾并在标签后追加 `*`。
	- 可累计记录多个不同临时路径。
	- 该记录仅会话内有效，不持久化；下次打开自动清空。
*   ✅ 已将“导入外部文件夹”按钮移到工程资源标题行右侧，不再占用底部提示栏。
*   ✅ 已支持工程资源与外部资源之间的左右拖拽分隔：
	- 可通过拖动中间分割条决定左右两侧的渲染空间。
	- 分割宽度会记忆到本地，下次打开可恢复。
*   ✅ 已支持面包屑自适应裁切：
	- 当标题行空间不足时，会从左侧逐级隐藏过长路径，仅保留更靠后的目录层级。
*   ✅ 已新增资源管理器最右侧“功能区”：
	- 功能区与左右资源区保持同一行，宽度随按钮内容自适应。
	- “导入外部文件夹”已迁入功能区，方便后续继续扩展更多按钮。
*   ✅ 已接入目录实时监听与增量刷新：
	- 主进程仅监听当前展示目录（工程/外部各一条监听），目录切换时自动解绑旧监听。
	- 监听事件会做短时间防抖与差量计算（add/remove/update），仅推送变更项到渲染层。
	- 渲染层按变更项更新列表，避免每次磁盘变化都全量重读目录，提升大目录场景下刷新性能。
	- 已修复 iframe 场景下监听事件投递：优先回发到发起监听的 frame，避免仅主 frame 收到事件导致资源区不刷新。

## 2. 核心架构与机制新计划 (Phase 5 进阶设想)

### 2.1 辅助对其机制 (Snap Guidelines)
*   为了支撑精细化视觉布置，在 Gizmo 层实现靠近其他同级节点或父节点边界时的“磁吸”效果。
*   提供中心线（X中心，Y中心）与边缘对齐参考线的高亮绘制。

### 2.2 多选与组级编辑 (Multi-Selection & Batch Editing)
*   **矩形框选**：中键/模式按键触发在 Viewport 画布内拉出矩形高亮区域，批量选中所覆盖节点。
*   **统一的 Inspector**：当选中多个支持同类属性配置的 Node 时，Inspector 可输入新属性并一次性更新它们（如改变选中3个节点的 Y 坐标）。

### 2.3 嵌套矩阵系(Transform Hierarchy)
*   如果 Scene 数据具有深层次层级，那么父子坐标计算就是必须解决的核心挑战。
*   未来重构 `core/Renderer.ts` 与交互 HitTest 时，需要引入完善的本地坐标至世界坐标转换矩阵系统（Local to World Matrix）。目前的拖拽逻辑在嵌套层内可能会因累计缩放和坐标系漂移而难以准确计算位移增距。

## 3. 未来扩展与建议 (New Suggestions)

### 3.1 基于 Plugin 的组件扩展机制
当前 `Renderer.ts` 处理各类 Laya 节点使用的是内部判断结构。如果 Laya 本身存在各种自定义类型/扩展组件（比如游戏内自制逻辑组件），应当从长远角度设计注册机制：
*   开放 `registerComponent(type, renderFn, inspectorSchema)` 方法。
*   不仅注册其外观渲染逻辑，连同该组件需要在右侧出现的专属动态表单 Schema 一并注册。

### 3.2 命令模式(Command Pattern)与全局快捷键分发
*   当前快捷键处理被包裹在了高层级 React 组件中。建议抽离 `ShortcutManager`，通过提供上下文提供键位绑定分发；
*   所有执行项（如 Delete, Clone, Save）转化为独立 Command，由命令总线发送给 Zustend 记录操作并生效，为完全稳定的 “全栈 Undo/Redo” 提供支持。

### 3.3 增量文件保存补丁 (Incremental Diff Saving)
*   目前是 `JSON.stringify` 完整覆盖写回，当未来 scene 文件过大时会带来卡顿、同时不友好于 Git 共同编辑。
*   **方案**：在操作中产生 Patch Action，在保存时加载原文件 JSON 结构字符串作 Diff 替换，能够使版本管理更干净。
