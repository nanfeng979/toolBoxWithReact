# 图片替换与比对工具 (Image Replacer) 开发计划

## 1. 目标与需求概述
开发一个新的 Mini App，主要用于左右两侧文件夹中图片的比对和定向替换。
- **布局**：顶部操作栏（包含多个功能按钮），下方为左右等分平齐的两个视图。
- **核心功能**：
  1. 左右视图均可加载本地文件夹，渲染所有图片并在图像下方显示文件名。
  2. 支持在左右视图各选中一张图片。
  3. **替换功能**：点击按钮后，将右侧选中的图片内容覆盖至左侧选中的图片文件，**但保持左侧文件名不变**。
  4. **排序与高亮功能**：点击按钮后，找出左右视图中同名的图片并将它们提取到列表的最前方，同时将同名图片的文件名文字渲染为绿色，方便快速定位。

## 2. 界面布局 (UI Layout)
- **Top Bar (顶部操作栏)**
  - `加载左侧文件夹` (Load Left)
  - `加载右侧文件夹` (Load Right)
  - `排序并高亮同名文件` (Sort & Highlight)
  - `执行替换` (Replace Left with Right)
  - *建议新增*：`撤销/恢复` (Undo) 或 `备份开关`
- **Main Content (主体内容区)**
  - **Left View (左侧视图, 50% 宽度)**：网格/列表展示图片缩略图+文件名。支持点击选中（选中后有边框高亮）。
  - **Right View (右侧视图, 50% 宽度)**：同上。

## 3. 技术选型与数据结构
- **前端框架**：React + TypeScript + TailwindCSS (根据现有环境)。
- **文件操作**：依赖工具箱提供的 Electron 主进程 API (如 `fs.readDir`, `fs.copyFile`, `fs.readFile` 等) 来读取图片和实施文件替换。
- **状态管理**：
  - `leftFiles`: 记录左侧文件列表 `{ path, name, isImage, url }`。
  - `rightFiles`: 记录右侧文件列表。
  - `leftSelected`: 当前左侧选中的文件路径。
  - `rightSelected`: 当前右侧选中的文件路径。

## 4. 优化建议与扩展计划 (Suggestions)
在核心需求基础上，提出以下建议以提升体验和安全性：
1. **防止误操作 (安全机制)**：由于涉及文件物理覆盖，建议在替换发生前自动在临时目录或同级目录下创建原文件的备份（例如改为 `filename.bak`），并提供一个一键还原的功能。或者是提供一个二次确认弹窗。
2. **拖拽加载 (Drag & Drop)**：除了通过按钮选择文件夹，支持直接将操作系统的文件夹拖拽到左右视图区域来快速加载。
3. **图片大图预览**：增加双击图片放大查看细节的功能，确保要替换的源文件和目标文件是正确的。
4. **支持的格式过滤**：读取文件夹时不仅加载图片，但也允许过滤功能（只显示 `.png`, `.jpg`, `.webp` 等），忽略系统隐藏文件。
5. **快捷键支持 (Hotkeys)**：可以使用左右方向键浏览图片，按 `Enter` 快速执行替换动作。
6. **批量同名替换 (Auto Sync)**：考虑到需要将同名文件排在前面，可能用户有“将右侧所有同名更新到左侧”的潜在批量需求。可以增加一个【一键替换所有绿色同名图片】的进阶按钮。

## 5. 开发步骤 (Milestones)
- **Phase 1**: 创建小程序骨架 (`app.json`, `index.html`, `App.tsx`)，并在 `appStore.ts` 中注册（如有需要）。
- **Phase 2**: 实现 UI 布局，切分上下结构与左右等宽双栏结构，绘制 TopBar 和图片 Grid 组件。
- **Phase 3**: 接入文件读取逻辑，实现加载文件夹与图片预览展示。
- **Phase 4**: 实现选中逻辑与“按同名重新排序并绿色高亮”的核心算法。
- **Phase 5**: 实现“替换”功能调用（结合 Node.js `fs.copyFileSync` 或相关的 IPC 通信机制），并加入相应的成功/失败提示。
